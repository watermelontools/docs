# Post Comment On Issue

Note: this will prepend a "üçâ" to the text.

## Route

```
api/gitlab/postCommentOnIssue
```

## Method

POST

## Params

```
{
    "issue_iid": "string",
    "owner": "string"
    "project_name": "string",
    "user": "string",
    "comment_body": "string"
}
```

Where:

- user: email of the user to search (registry on app). Not necessarily GitHub email.
- owner: owner of the git repo
- project_name: the name of the project the issue belongs to.
- issue_iid: the iid for the issue, not the id.
- comment_body: body of the comment to add, allows markdown.

## Response

### Errors

If the user does not exist, there will be no access token

```
{
    "error": "no access_token"
}
```

If nothing comes in the `user` param

```
{
    "error": "no user"
}
```

If nothing comes in the `owner` param

```
{
    "error": "no owner"
}
```

If nothing comes in the `project_name` param

```
{
    "error": "no project_name"
}
```

If nothing comes in the `issue_iid` param

```
{
    "error": "no issue_iid"
}
```

If nothing comes in the `comment_body` param

```
{
    "error": "no comment_body"
}
```
