# Get Most Relevant Jira Ticket

## Route

```
api/jira/getMostRelevantJiraTicket
```

## Method

POST

## Params

```
{
    "user": "string",
    "prTitle": "string"
}
```

Where:

- _user_: email of the user to search (registry on app). Not necessarily Jira email.
- _prTitle_: Title of the most relevant PR or fallback to latest commit. It will search for hotwords.

## Response

### Errors

If the user does not exist, there will be no access token

```
{
    "error": "no access_token"
}
```

If nothing comes in the `user` param

```
{
    "error": "no user"
}
```
